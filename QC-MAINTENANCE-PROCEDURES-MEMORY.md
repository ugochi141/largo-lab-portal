## QC Failure Actions

### ‚ö†Ô∏è Critical QC Rule

**NEVER report patient results if QC is not acceptable.**

Patient safety depends on accurate results. If QC fails:
1. Troubleshoot and repeat
2. If still failing, use backup method or send to reference lab

### QC Failure Troubleshooting Steps

When QC fails, follow these steps IN ORDER:

1. **Repeat QC analysis**
2. **Check reagent expiration dates**
3. **Inspect instrument for problems**
4. **Contact manager if QC continues to fail**
5. **Do NOT report patient results until QC is acceptable**

### Follow-up Notification Requirements

1. **Immediately notify Lead Technologist, TQS, and management** of QC failure. Proceed to troubleshooting.
2. **Document all troubleshooting steps attempted**
3. **Contact Tech Support and obtain reference number**
4. **Notify Technical Quality Specialist with:**
   - All troubleshooting steps performed
   - Tech support reference number (add to Handoff Sheet)
   - Current instrument status
5. **Notify Laboratory Manager** with complete documentation

**CRITICAL:** Before contacting Technical Quality Specialist and Lab Manager, ensure you have:
- Documented all troubleshooting steps
- Obtained the reference number from Tech Support

**DO NOT leave it for next shift** - must troubleshoot and contact tech support, then pass down to next shift.

## ‚ö° QC Failure Action Plan Algorithm

### Step-by-Step Response to QC Failure

**Step 1: STOP**
- Do not report patient results

**Step 2: EVALUATE**
Determine error type:
- **Random error** ‚Üí Repeat QC immediately
- **Systematic error** ‚Üí Proceed to troubleshooting

**Step 3: TROUBLESHOOT**
Follow systematic approach:
- Check for obvious issues (reagent expiration, probe clogs)
- Review recent changes (maintenance, reagent lots)
- Run fresh control material
- Check calibration status

**Step 4: DOCUMENT**
Record all actions:
- QC failure details
- Troubleshooting steps performed
- Root cause identified
- Corrective action taken

**Step 5: VERIFY**
Confirm resolution:
- Run QC to verify acceptable performance
- If using backup method, verify correlation

**Step 6: NOTIFY**
Communication requirements:
- **Lead Technologist** - Immediately
- **Technical Support** (if needed) - Get reference number
- **Technical Quality Specialist** - With documentation
- **Laboratory Manager** - Complete report

**Step 7: RESUME**
Return to testing:
- Only after QC passes
- Review any held patient samples
- Consider repeat testing if indicated

## Pre-QC Workflow Steps

**This workflow applies to:**
- Pure Analyzer 1
- Pure Analyzer 2
- All QC runs
- All calibrations
- All maintenance procedures

**Remember:** Print ‚Üí Verify ‚Üí Run ‚Üí Document

### Step 1: Print QC Load List

Before running QC, calibration, or maintenance on either Pure Analyzer, you **MUST first print out a QC Load List**.

### Step 2: Navigate to Pre-Routine

Access the Pre-Routine menu and check the following items in Reagent Management List:

- ‚úì Download Required Items
- ‚úì Sample Records Management
- ‚úì Recommend Calibration
- ‚úì Routine QC
- ‚úì Standby QC (New Lot Only)
- ‚úì Calibrator and QC Load List
- ‚úì Update SW Components

### Step 3: Print Routine QC

In the Routine QC section, ensure you print the report.

### Step 4: Calibrator and QC Load List

- Check both boxes
- Select "Confirm"

### Step 5: Verify Reagent Lots

Once the QC Load List is printed:

1. Compare the reagent lots on the analyzer to the lots listed on the printout
2. Check if reagents need a new bottle or are a new lot
3. **IMPORTANT:** Notify the Lead Tech or Shanti immediately about any lots that are in use but not entered in the system

### Step 6: Documentation

**MANDATORY:** Attach the QC Load List to the shift handoff form

## Quick Reference

### Q: What do I do if QC fails?

**A:** Follow the 7-step QC Failure Action Plan:
1. STOP (don't report results)
2. EVALUATE (random vs systematic error)
3. TROUBLESHOOT (systematic approach)
4. DOCUMENT (all steps)
5. VERIFY (run QC again)
6. NOTIFY (Lead, Tech Support, TQS, Manager)
7. RESUME (only after QC passes)

### Q: What must I do before running QC on Pure analyzers?

**A:** Print QC Load List ‚Üí Verify reagent lots ‚Üí Run QC ‚Üí Attach to handoff form

### Q: What if MOB Pure 1 fails QC before 11am?

**A:** Redirect Chemistry testing to AUC immediately. Mark log with `*` and document.

### Q: Who do I notify for QC failures?

**A:** In order:
1. Lead Technologist (immediately)
2. Tech Support (get reference number)
3. Technical Quality Specialist (with documentation)
4. Laboratory Manager (complete report)

### Q: Can I skip QC if I'm busy?

**A:** Absolutely not. Zero tolerance for skipping, faking, or forging QC. This is a serious compliance violation.
